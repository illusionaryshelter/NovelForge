
# @Programmer(程序员)
## 项目需求
我目前在进行一个软件的编写，这个软件是用于写网络小说使用，提供GUI，包括时间线管理，人物管理，世界观管理，剧情管理，章节管理等。能够根据时间线点击跳转到人物当前剧情，不同角色拥有根据阵营分类功能，并且能够根据阵营筛选人物。剧情管理能够根据时间线和人物进行筛选，并且能够根据人物进行筛选。可以插入新角色，以及让角色的动机随着时间线而变化，从而不同角色之间产生交互。



## 调试与开发规则（严格遵守）
1. **接口友好**： 无需使用复杂的设计模式，而是选择便于可读性的，充分考虑性能以及良好的注释
2. **防御性编程**：做好各种越界和exception检查，便于调试。
3. **构建可视化**：如果使用了c++，在生成 `CMakeLists.txt` 时，默认开启 `set(CMAKE_VERBOSE_MAKEFILE ON)`，确保在编译报错时能看到完整的链接信息，方便调试。
4. **渐进式开发**：禁止一次性生成整个项目的代码。必须按模块推进。
5. **模块编程**：模块之间低耦合，便于与其他项目进行快速合并，需要做好单元测试
6. **跨平台**：当前平台是aarch64-linux，但是尽量确保Windows下也可以使用，并且最好是直接打包成.exe+.dll
